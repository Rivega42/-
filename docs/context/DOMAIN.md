## Физическая структура шкафа

### Шкаф (Cabinet)
Физическое устройство целиком — корпус, моторы, ридеры, RPi.

### Ряд (Row)
Один из двух уровней вдоль глубины: `FRONT` (передний, читателю ближе) и `BACK` (задний).

См. `bookcabinet/config.py` → `Row.FRONT`, `Row.BACK`.

### Колонка (Col)
Вертикальная позиция по горизонтали. 3 колонки (0, 1, 2). Колонка 1 — центральная, где находится окно выдачи.

### Позиция (Position)
Высота полки (Y координата). 21 позиция (0..20), нумерация снизу. Позиция 9 — окно выдачи.

### Ячейка (Cell)
Одна логическая «ячейка» = (row, col, position). Всего 2 × 3 × 21 = **126 ячеек**.
Адресуется обычно `(col, row, pos)` или просто `Address`.

### Полочка (Shelf)
Физический поднос, который двигается тарелка/каретка.
В коде встречаются термины `shelf`, `tray`, `платформа` — не всегда строго различаются.

### Каретка (Carriage)
Подвижная часть, которая ездит по XY (CoreXY кинематика). Несёт на себе тарелку.

### Тарелка / Tray Platform
Платформа на каретке, куда садится полочка для перемещения. Двигается вверх/вниз отдельным мотором (`TRAY_*`).

### Окно выдачи (Issue Window)
Физическое отверстие в корпусе на высоте позиции 9, колонка 1, ряд FRONT. Через него читатель видит/берёт книгу.
В коде: `WINDOW_POSITION` или `{row: FRONT, x: 1, y: 9}`.

### Шторки (Shutters)
Релейные заслонки, перекрывают/открывают окно выдачи. Есть **внешняя** (наружу) и **внутренняя** (внутрь шкафа). См. CLAUDE.md §5.6.

### Замки (Locks)
Серво-приводные язычки, фиксируют полочку на каретке. См. GLOSSARY про инверсию open/close.

### Endstops
Концевые датчики на крайних позициях:
- `SENSOR_LEFT`, `SENSOR_RIGHT` — XY горизонталь.
- `SENSOR_TOP`, `SENSOR_BOTTOM` — XY вертикаль.
- `SENSOR_TRAY_BEGIN`, `SENSOR_TRAY_END` — тарелка.

### HOME corner
Положение каретки в начале / после homing. Canonical: **LEFT + BOTTOM** (см. CLAUDE.md §8).

## Бизнес-сущности

### Читатель (Reader)
Пользователь библиотеки. Идентифицируется по:
- **Читательский билет** — карта библиотеки, NFC.
- **ЕКП** (Единая Карта Петербуржца) — городская карта, NFC.

### Книга (Book)
Физический экземпляр. Идентифицируется по UHF RFID метке.
В ИРБИС у книги есть инвентарный номер, шифр, библиографическая запись.

### Выдача / Issue
Процесс: читатель прикладывает карту → шкаф находит книгу → выдаёт через окно → ИРБИС фиксирует выдачу.

### Возврат / Return
Процесс: читатель кладёт книгу в окно → шкаф проверяет UHF → принимает на полку → ИРБИС фиксирует возврат.

### Бронь / Reservation
Книга, заказанная читателем заранее в ИРБИС. Шкаф должен её выдать конкретному читателю.

### Инвентаризация / Inventory
Плановый процесс: шкаф объезжает все ячейки, читает UHF метки, сверяет с ИРБИС.

## Системные термины

### Bridge (Python ↔ Node)
`bookcabinet/bridge.py` — гейт для команд от Node-сервера к Python-логике.

### Daemon
Долгоживущий процесс на Pi, запускаемый systemd. Главный — `bookcabinet-daemon.service`.

### Mock mode
Режим без реального железа (для разработки на ноуте). Активируется когда `pigpio` не доступен.

### Calibration
Процесс измерения реальных X/Y координат каждого ряда/полочки. Сохраняется в `calibration.json`.

### Position sync
Сохранение последней известной позиции каретки между перезапусками (чтобы не делать homing каждый раз).

### Homing
Процедура поиска физического HOME corner движением до endstops. См. `tools/homing_pigpio.py`.

### Direction safety
Защита: при движении в сторону endstop X — мы уже на этом endstop'е → не двигаемся. См. `tools/corexy_pigpio.py` `DIR_TO_SENSOR`.

### Glitch filter
pigpio функция `set_glitch_filter` — игнорирует кратковременные изменения сигнала на endstop. Защита от шума.

### Wave chain
pigpio механика для длинных последовательностей шагов мотора. Используется в `corexy_motion_v2.py`.

### Belt slip
Проскальзывание ремня привода CoreXY → пропуск шагов → потеря позиции. Mitigation: умеренные скорости (800/300).

## Языковые конвенции

- **Код / идентификаторы:** только латиница, snake_case (Python) или camelCase (TS).
- **UI текст для читателя:** русский, простой язык.
- **UI текст для оператора:** русский с допустимыми терминами (RFID, ИРБИС, ЕКП).
- **Логи / комменты в коде:** английский.
- **Документация:** русский основной, английские термины inline.
- **Branch / commits:** английский (Conventional Commits).
