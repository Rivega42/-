# Product Context — BookCabinet

> Бизнес-контекст продукта для AI-исполнителей. Что такое BookCabinet, для кого, зачем.

---

## Что такое BookCabinet

BookCabinet — **автоматизированный библиотечный шкаф**. Физический терминал, установленный в публичной библиотеке, который позволяет читателям самостоятельно:
- брать книги (issue),
- возвращать книги (return),
- идентифицироваться через карту ЕКП или библиотечный билет,
- получать обратную связь через kiosk UI на touch-screen.

Внутри — механический шкаф с полками и приводом, RFID/NFC ридеры, Raspberry Pi с web kiosk UI, интеграция с ИРБИС (библиотечная информационная система России).

## Для кого

### Конечные пользователи (читатели)
- Возраст: 7-90 лет.
- Технический уровень: разный, в т.ч. пожилые люди без опыта с touch-интерфейсами.
- Время взаимодействия: короткое (1-3 минуты).
- Контекст: пришли в библиотеку, хотят взять/вернуть книгу.

### Операторы (библиотекари)
- Получают уведомления о работе шкафа (Telegram).
- Решают edge cases (jam, unknown card).
- Проводят инвентаризацию (плановую через UHF batch read).
- Без них шкаф работает 95% времени.

### Founders / разработчики
- Roman — owner, hardware engineer, on-site.
- Контрибьюторы — software dev, AI-агенты.

### Заказчик
- **Зеленогорская библиотека** (СПб) — пилотный заказчик, Q2 2026.
- Дальше — другие библиотеки в сети.

## Зачем

### Проблема, которую решаем
- Библиотекари перегружены рутиной выдачи/возврата.
- Часть читателей хочет сервис вне рабочих часов библиотекаря.
- Учёт книг в ИРБИС часто отстаёт от реальности.
- Молодёжь привыкла к self-service (бургер-кинг, метро, маркетплейсы) и ожидает того же от библиотек.

### Value proposition
- Для **библиотеки**: меньше рутины, больше свободного времени персонала, точный учёт.
- Для **читателя**: быстрее, доступнее, можно даже когда библиотекарь занят.
- Для **города/министерства**: модернизация библиотечной инфраструктуры.

## Бизнес-модель (текущая стадия)

- Q2 2026: пилот в одной библиотеке. Бюджет — частично грантовый / частично из библиотеки.
- Q3-Q4 2026: тираж в 2-3 библиотеки. Цена — TBD по итогам пилота.
- Q1 2027+: масштабирование сети. Возможны варианты: продажа шкафов / SaaS / lease.

## Конкуренты / аналоги

- Импортные library kiosks (3M, EnvisionWare) — дорогие, не интегрированы с ИРБИС.
- Российских аналогов мало или нет.
- DIY решения отдельных библиотек — самописные, не масштабируемые.

## Конкурентное преимущество

- Нативная интеграция с ИРБИС.
- ЕКП поддержка (СПб специфика).
- Open architecture, расширяемость.
- Стоимость ниже импортных аналогов.
- Локальная поддержка.

## Регуляторные ограничения

- **152-ФЗ** (персональные данные) — обработка данных читателей.
- **Закон о библиотечном деле**.
- Совместимость с ИРБИС (стандарт МАРК-21 / RUSMARC).
