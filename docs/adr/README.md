# ADR — Architecture Decision Records

В этой папке — формальные ADR'ы по проекту. Каждое решение — отдельный файл.

## Формат имени файла

`NNNN-краткое-имя.md` — например `0001-corexy-home-corner.md`.

## Шаблон

```markdown
# ADR-NNNN: <Название>

- **Дата:** YYYY-MM-DD
- **Статус:** proposed | accepted | superseded by ADR-XXXX | deprecated
- **Авторы:** @nick1, @nick2

## Контекст
Почему этот вопрос вообще встал. Что мы знали на момент решения.

## Решение
Что именно решили сделать. Конкретные технические выборы.

## Последствия
- Что становится проще/возможным.
- Что становится сложнее/запрещённым.
- Какие риски остаются.

## Альтернативы
Что рассматривали и почему отклонили.

## Связанные документы
- Issues, PR, более ранние ADR.
```

## Связь с другими DECISIONS-файлами

- `DECISIONS.md` (root) — короткие записи решений (1 параграф).
- `docs/DECISIONS.md` — старые hardware/business решения, исторический формат.
- `docs/adr/*.md` — детальные ADR с альтернативами и последствиями.

При новом важном решении: создаётся файл в этой папке + ссылка из `DECISIONS.md` (root).
