# _archive/

Сюда **перемещаются** удаляемые файлы вместо `git rm`.

## Зачем

Проект эволюционировал на живом железе. Многие "устаревшие" файлы хранят знания, которые ещё могут пригодиться при отладке. См. `CLAUDE.md` §11.5.

## Конвенции

- Структура: `_archive/YYYY-MM-DD/<original-path>`.
- Каждая дата = отдельная "пачка" архивирования.
- В корне даты-папки — `README.md` с пояснением: что и почему перемещено.

## Пример

```
_archive/
  2026-04-30/
    README.md            # объясняет: чистка после repo-setup
    docs/old_TODO.md     # перенесли потому что мигрировали в Issues
    tools/legacy_x_homing_v1.py
```

## Когда удалять окончательно

Через ≥3 месяца после архивирования, если файл не пригодился. Удаление окончательное — отдельный PR с явным согласием founders.

## Не клади сюда

- Секреты (даже устаревшие — лучше в private gist + ротация).
- Огромные бинарные файлы (используй git LFS или внешнее хранилище).
